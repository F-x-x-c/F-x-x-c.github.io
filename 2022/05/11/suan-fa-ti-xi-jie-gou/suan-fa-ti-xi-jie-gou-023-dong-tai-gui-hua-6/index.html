1. 数组分裂和

  1. 题目

    给定一个正数数组arr, 把正数数组分为两个集合, 并且尽量让两个集合的累加和接近, 返回小的那个集合的累加和;

  2. 思路
    递归函数的定义: 对于arr[i… arr.len - 1]这个区间, 区间的累加和为sum, 求出这个区间上的接近但不大于sum / 2的元素选择累加和; 这需要对该区间上的每一个位置的元素都进行选择, 选择要还是不要; 而判断的依据需要依赖这个位置和下一个位置的选择情况, 需要抓取到下一个arr[i - 1… arr.len - 1]上的结果, 然后与选择了这个位置的情况作比较, 返回较大值即可;

  3. 是否有利可图?
    对于数组[5, 2, 3…], 要凑20; √××, 那么就是(i = 4, rest = 15); ×√√, 那么就是(i = 4, rest = 15), 有利可图;

2. 数组分裂和.限制个数版

  1. 题目
    给定一个正数数组, 如果arr的元素个数为偶数, 则要求两个集合的包含的个数一样多, 如果为奇数, 则两个集合的元素个数必须只差一个; 在这种情况下, 尽量让两个集合的累加和接近; 返回小的那个累加和;

  2. 思路
    只需要按照上面的题目的思路, 加上一个限制条件restNum即可; 但是实现起来却有点麻烦; 尤其是左神的思路;

  3. 解法1: 递归
    根据题意, 如果arr元素个数为奇数的话, 则需要比较两个累加和, 返回较小的那个累加和, 也就是说, 较小的累加和有可能是元素个数多的那个集合, 也可能是小的那个集合; 递归函数定义: 在arr[i … len - 1]上进行自由选择, 挑选的个数一定要是restNum个, 累加和小于等于rest;

    具体实现:

    (1) 当遍历到末尾的时候, 如果说挑选的刚好挑选完毕, 也就是restNum为0, 则为有效解, 否则为无效解; 为有效解时, 考虑递归函数的定义, 一定返回0, 也就是在一个没有数的领域去挑数, 以此去凑rest, 但是因为没有数了, 所以这个范围就应当返回0; 而为无效解时, 约定返回1;

    (2) 从左至右的选择模型, 递归函数中对所操作的位置进行选择, 所以有两种情况, 选和不选; 不选很简单, 但是选就有点复杂; 根据题意, 这个集合的总和不能超过rest, 所以如果正在操作的位置index对应的数比rest大, 那么就为无效解, 反之就为有效解, 需要先通过递归函数获取到递归的结果, 如果这个递归结果仍然是-1, 意味着也为无效解, 如果不是1, 则需要结果需要加上这个位置的值; 将其与不选这个位置的结果进行比较, 返回较大值即可;

  4. 解法2: 动态规划

    需要使用三重数组, 不需要太过于关注挑选的个数为奇数还是偶数, 不过在初始化的时候, 其他的倒是没什么;

3. N皇后问题

  1. 题目
    给定一个N * N的棋盘, 依次将N个皇后放在棋盘上, 要求皇后之间两两不能同行、同列和同斜线, 返回有多少种摆法;

  2. 思路1: 递归

    在棋盘的第一行第一个位置摆一个皇后, 然后第二行第三个位置再摆一个, 每一行都摆一个皇后, 如果这一行的皇后摆在哪里都不合适, 那就调整上一行的皇后;
    该题不可用从前简单的对位置依赖进行分析而得出的动态规划, 而是使用更高端的技巧;

  3. 解法1

    一共要处理N个皇后, 递归函数定义: 在第index行进行皇后的摆放, 要求不能够与前面的皇后起冲突, 如果没有起冲突, 那就把这个位置信息扔到下一层的递归函数进行方法数的获取; 递归函数有3个参数, index表示在第几行进行皇后的摆放, 一个整型的数组before, 该数组用于存放已经放置的皇后的列数, 因为用到这个数组的地方只有两个, 一个是去判断是否冲突, 这时需要进行数组从头开始的遍历, 所以不需要行数, 另一个地方是紧接着之后, 如果这个位置与之前所有的皇后都不冲突, 则把这个位置的列数放在下一层的递归函数中去, 进行结果的获取, 而这里因为是遍历的index这一行的每一个位置(每一列), 所以在进行before数组的填写时也可以直接获取行数, 综上, 存储前面的皇后的信息只需要存储这些皇后的列数即可; 判断是否冲突: 遍历before数组, 由于前面的皇后的行数绝对不超过当前递归函数的index, 所以只需要从0遍历到index, 如果出了一点错误, 直接返回false; 而判断的条件由于已经不同行, 那么只需要不同列以及不同斜线即可, 不同列很简单, 而不同斜线需要拿到比较的两个位置的坐标, 横纵坐标分别相减, 再进行相等判断即可;

  4. 解法2: 优化常数项时间的解法

    由于N皇后问题里N的数值不宜过大, 所以可以采用整型值的位, 来存储信息, 而在N皇后中, 具体的信息就为已选择的皇后对后面皇后的选择的影响; 这里需要重新审视解法1, 因为每一行只可能有一个皇后, 所以对行进行遍历, 选择皇后, 而前面选择的皇后对后面皇后的选择有影响, 已选取的一个皇后对某行的皇后选取的影响为, 皇后的左斜线右斜线和竖直线与该行的相交元素不可选; 而从整个遍历过程来考虑, 第一行所选择的皇后对第二行的影响为这个皇后的左下, 下, 右下位置不可选, 而对第三行的影响为刚刚提到的位置里的左下位置的左下位置, 以及刚刚提到的位置的右下位置的右下位置不可选, 所以这可以采用三个整型变量来存放这些限制, 分别为左下限制、列限制和右下限制。而由于是一行一行的遍历，并且一个已选皇后的左下和右下的影响按照斜线递增，所以在传递左下限制和右下限制时可以对左下右下限制对应的整型变量采用左移和右移位数据的方式；

  5. 解法2：递归函数详解

    传递参数：
    int limit：该整型变量的后N个位全为1，也就是coLimit的最终形态，负责与coLimit完成终止条件的判断；
    int coLimit：某个皇后的列限制，最终形态为limit；
    int leftLimit：某个皇后的左下对角线限制；
    int rightLimit：某个皇后的右下对角线限制；

    终止条件：当coLimit等于limit时，返回1；也就是说，由于要摆N个皇后，所以必定是每一列摆一个，所以列限制的终止条件必定为limit，如果等于了，则表明找到了一种有效方法，所以返回1；

    递归本体：首先要使用一个变量找出所有可以尝试的位置，用1表示, 原因是表示为1之后比较好挨个拿1; 由于那些limit的位为1时表示不可选, 为0时表示可选, 把他们或起来, 再取反后的结果为: 为1的位是可以尝试的位, 但是高出N位的那些位也为1; 所以需要将这个结果与limit相与, 使得高出N的位为0, 又不影响到低N位, 这样就可以得到正确的用1表示可尝试位置的变量canTry了;
    然后就是挨个尝试可以尝试的位置了; 这里涉及到获取一个变量最右边的1的技巧: 我们获取的是变量canTry最右边的那个1, 这个1如何表示呢? 肯定是要用一个变量来存储的, 所以工作是用一个变量mostRightOne来存储变量canTry最右边的1, 而注意, 对于最右边的那个1(这个位不妨叫它mro), 它的右边一定全部都是0, 那么将变量canTry取反, 使得mro变为0, 而mro右边的位都变为了1, 取反后得到的结果加1, mro右边的位会不断地往左边进位, 使得mro又变回了1; 而对于mro左边的位, 由于加1的操作并没有影响到它们, 加1的影响到了mro就停止了, 所以mro左边的位就为取反的结果, 然后再把这个结果和canTry相与, 就得到了想要的那个变量mostRightOne, 其实就一句代码的事:
