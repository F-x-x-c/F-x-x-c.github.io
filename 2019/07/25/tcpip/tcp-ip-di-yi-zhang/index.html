1.2 设计和实现

1.2.2 复用 分解和封装

协议复用允许相同协议对象的多个实例同时存在, 类似于面向对象编程中的类和对象的关系;

每一层的对象被称为协议数据单元(PDU), 在传递到下面一层时, 下面一层会将不会过问传递下来的数据, 把它当作无需解释的信息, 封装上这一层该有的信息头部, 然后传递到下一层;

用户主机和交换机分别被称为端系统和中间系统, 交换机和网桥并不能成为中间系统, 因为它们不具有IP地址, 只是起到一个转发的作用; 有时候路由器和交换机由于需要满足远程操控等管理方面的功能, 也需要实现传输层和应用层;

1.3 TCP/IP协议族和协议

1.3.1 arpanet参考模型

地址解析协议(ARP)用于完成IP层的地址和链路层使用的地址的转换, 是IPv4的专用协议;

IP协议发送给链路层协议的PDU被称为数据报, IPv4为64KB, IPv6为4GB; 数据报太大的话, 需要在被链路层包装为它的PDU时进行分片;
IP地址的类型决定如何转发数据报, 单播为目的地为一台主机, 多播(广播)的目的地为一个指定的网络, 组播的目的地为一个组播组中的一组主机; 其实就是目标主机不同而已;

Internet控制消息协议(ICMP)被分到3.5层, 用于主机与其他主机或路由器的IP层之间交换差错消息和其他重要消息; ping和traceroute使用的协议就是ICMP;
Internet组管理协议(IGMP)采用组播寻址和交付来管理组播成员主机;
ICMP和IGMP都为IP辅助协议;

在第4层, 传输控制协议(TCP)采用面向连接, 不处理消息边界的方式, 可以处理数据包丢失, 重复和重新排序等IP层不处理的问题;
而用户数据报协议(UDP)仅提供比IP协议稍多的功能, 允许应用发送数据报, 保留消息边界, 不强制实现速率控制和差错控制;
对比: TCP通过处理一系列的例如丢包、重排等问题实现数据报的可靠传输, 使用TCP的应用可以无需再考虑可靠性的问题; UDP仅提供一套用于复用、分解数据和校验数据完整性的端口号, 人和可靠性都需要应用层提供;
