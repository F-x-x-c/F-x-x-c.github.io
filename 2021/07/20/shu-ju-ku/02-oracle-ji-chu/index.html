3. order by排序

select * from emp order by sal; –以工资进行升序排列

select * from emp order by sal desc; –以工资进行降序排列; desc为降序, asc为升序

select * from emp order by ename; –以字符进行排序时, 是以字典序进行的排列;

每次在执行order by的时候相当于是做了全排序, 思考全排序的效率; 会比较耗费资源, 因此需要在选择业务不太繁忙的时候进行, 比如凌晨;

4. 字符连接符

select ‘my name is ‘ || ename name from emp;

select concat(‘my name is ‘, ename) from emp;

5. 使用计算字段

null是比较特殊的存在, 和Java不同, Java的null都是指的一个内存空间, 而oracle的null和null之间各不相同, 而且任何值和null进行运算, 得到的还是null;

dual是oracle中的测试用表
select 100+null from dual;

select ename, (e.sal+nvl(e.comm,0))*12 from emp e;

nvl(arg1, arg2)表示如果arg1等于空, 那么返回arg2, 如果不等于空, 那么返回arg1;

6. 交并差补集

  1. 并集

    将两个集合进行合并, 显示所有的元素, 但不包含重复的数据
    select * from emp where deptno=10 union
    select * from emp where sal > 2000;

  2. 全集
    将两个集合的数据全部显示, 但是不进行去重操作;
    select * from emp where deptno = 30 union all
    select * from emp where sal > 1000;

  3. 交集
    两个集合中共有的数据
    select * from emp where deptno = 30 intersect
    select * from emp where sal > 1000;

  4. 差集
    包含在集合A但不包含在集合B中的数据, 也就是A - A∩B; 所以要特别注意顺序;
    select * from emp where deptno = 30 minus
    select * from emp where sal > 1000;
